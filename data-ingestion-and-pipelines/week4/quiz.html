How is orchestration different from simple Cron scheduling?

Correct

Unlike simple Cron scheduling, orchestration enables you to set up task dependencies so certain tasks don't start until others have successfully completed, monitor tasks, receive alerts on task failures and retry tasks.

Question 3

One of the Airflows core components is the scheduler. What is the role of the scheduler?

Correct

The scheduler is a critical component of Airflow. It constantly checks for tasks that are ready to be executed based on their schedule or the completion of their dependencies. The scheduler pushes ready tasks to a queue and uses an executor to manage their execution.

When you instantiate a DAG in Airflow, which parameter do you need to specify if you dont want the scheduler to start the DAG runs that have not been run since the last data interval?

Correct

The scheduler, by default, will start a DAG Run for any data interval that has not been run since the last data interval. If you dont want your DAG to handle its catchup, then you need to set catchup to False.
